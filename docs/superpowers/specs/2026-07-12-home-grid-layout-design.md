# YourBlog 首页网格布局设计

**日期：** 2026-07-12
**状态：** 已确认，待编写实施计划

## 目标

扩展“外观与页面 > 首页编辑”，使管理员无需编写代码即可控制首页区块的横向占比、视觉高度和行起点。现有的区块显隐、上下排序和文案编辑继续保留。

系统必须保持当前的通透玻璃视觉、中文 UTF-8 界面、Docker Compose 部署方式和 1C2G 服务器适配性。不引入拖拽式低代码框架、画布、任意 CSS 输入或运行时执行器。

## 范围

本次包含以下首页模块的布局控制：首屏介绍、精选教程、最近文章、网页实验室和教程归档。

每个模块可配置：

- **桌面宽度：** 全宽 `12/12`、宽 `8/12`、半宽 `6/12`、窄 `4/12`。
- **区块高度：** 内容自适应、紧凑、标准、舒展。
- **起新行：** 开启后模块从当前布局的第一列开始，不与前方模块填充同一行。

本次不包含任意坐标拖拽、像素输入、独立的平板/手机编辑、模块嵌套、自由组件插入和自定义 CSS/JavaScript。

## 后台交互

首页编辑页继续以稳定的模块列表作为入口。管理员点击某个模块后，展开该模块的布局检查器，显示：

1. 四档桌面宽度分段控件。
2. 四档高度分段控件。
3. “从新的一行开始”开关。
4. 当前配置摘要，例如“8/12 · 舒展 · 新行”。

模块的上移、下移和显隐按钮保持可用。布局检查器只负责模块占位和比例；首页标题、描述、标签、实验室文案和文章数量仍位于现有的内容字段区域。

右侧提供未保存状态的即时预览，并可切换桌面、平板和手机。预览显示真实模块名称、文案层级和文章卡片密度，但不写入数据库。管理员点击“保存首页”后才发布布局；恢复默认会重置模块顺序、显隐、布局和首页文案，且必须二次确认。

## 数据模型与兼容性

不新增数据库表或 Prisma 字段。现有 `HomeLayoutSettings.modules` JSON 中的每个模块扩展为：

```ts
type HomeModule = {
  id: "hero" | "featured" | "recent" | "labs" | "archive";
  enabled: boolean;
  layout: {
    desktopSpan: "full" | "wide" | "half" | "narrow";
    height: "auto" | "compact" | "standard" | "spacious";
    breakBefore: boolean;
  };
};
```

存量 JSON 没有 `layout` 时，读取层自动补为兼容默认值：`full`、`auto`、`false`。因此升级后的首页保持原有纵向全宽排列；管理员保存新的布局后才产生网格效果。

新配置由领域层验证：模块标识必须完整且不重复，尺寸和高度必须在允许集合中，`breakBefore` 必须为布尔值。无效配置不写入数据库，并在后台显示中文字段错误。

## 公开端渲染

公开首页使用受控 CSS Grid，不读取或执行管理员输入的 CSS：

| 设备 | 布局规则 |
| --- | --- |
| 桌面 | 12 列。`full=12`、`wide=8`、`half=6`、`narrow=4`。 |
| 平板 | 2 列。`full` 与 `wide` 占满两列，`half` 与 `narrow` 各占一列。 |
| 手机 | 单列。所有模块依顺序堆叠。 |

`breakBefore` 会让模块从每个断点网格的第一列开始。高度档位仅设置最小高度，不设置固定高度：内容自适应使用自然高度，紧凑、标准、舒展分别提供逐级增大的最小块高度。标题、摘要和文章卡片始终可以扩展，不允许被裁切或遮挡。

区块内容和现有链接行为不变：精选教程仍链接教程页，网页实验室仍链接实验室，归档仍链接归档页。排版配置只影响首页容器与模块的视觉占位。

## 模块边界

实现继续位于 `site-designer` 模块：

- 领域层定义布局令牌、默认值、兼容解析和验证。
- 应用层负责标准化配置并原子保存。
- Prisma 适配层读取和保存已扩展的 JSON。
- 后台 UI 维护本地预览状态；服务端操作只接收受控表单值。
- 公开首页组件通过模块公开接口取得已验证配置，并映射为 CSS 类名或变量。

不让公开组件读取后台表单状态，也不让后台 UI 直接访问 Prisma。现有文章、媒体、网页实验室和普通页面模块保持不变。

## 错误处理与恢复

- 保存前和服务端均验证布局，避免手工篡改表单绕过受限选项。
- 无法识别的旧 JSON 使用安全默认布局，而不是造成首页渲染失败。
- 恢复默认需要确认；不删除教程、媒体或普通页面数据。
- 保存失败时保留当前编辑状态并显示中文错误；公开首页继续使用上一次有效配置。

## 性能与部署

该功能只扩大一份小型 JSON 配置和按需加载的后台编辑代码。公开端继续服务端渲染，浏览器只执行轻量的预览状态更新。无新增容器、后台进程、大型依赖或媒体副本，Docker 镜像与运行内存预期仅有极小的业务代码增量。

## 测试与验收

- 单元测试：默认布局补全、旧配置兼容、非法宽度/高度/布尔值、重复模块和保存标准化。
- 组件测试：布局控制器的尺寸选择、换行开关、预览状态和恢复默认确认。
- 公开渲染测试：桌面跨度、平板两列、手机单列、起新行和高度不裁切。
- 回归验证：UTF-8、模块边界、Lint、类型检查、完整测试、Next.js 生产构建和 Docker Compose 健康检查。
- 人工验收：后台三端预览、保存后公开首页更新、现有教程与实验室链接、移动端文字与按钮不溢出。

## 成功标准

管理员能够在首页编辑页为每个模块安全地调整宽度、高度和行起点，并立即理解其在桌面、平板和手机上的效果。布局不允许重叠或裁切内容，旧首页配置兼容，现有博客功能和轻量部署特性不回归。
