# YourBlog 外观与页面管理设计

**日期：** 2026-07-12
**状态：** 已确认，待编写实施计划

## 目标

为不熟悉编程的管理员提供可视化的“外观与页面”后台模块。管理员无需修改代码，即可调整全站视觉参数和首页模块，创建、预览、发布普通内容页面。功能须保持当前 Apple 风格、通透玻璃质感和中文界面，并适配 1C2G 服务器与 Docker Compose 部署。

## 范围与边界

本次实现包含：

- 全局外观配置：主题、主色、背景层次、玻璃通透度、圆角、按钮样式与顶部栏密度。
- 首页配置：首屏介绍、精选教程、最近文章、网页实验室、教程归档等既有模块的可见性、顺序和允许编辑的文本或数量。
- 普通页面：标题、简介、封面、富文本正文、SEO、草稿/发布状态、访问地址和导航栏展示选项。
- 后台即时预览和普通页面的管理员预览。

本次不包含：自由拖拽布局、任意自定义 CSS/JavaScript、用户自定义组件、页面模板市场、版本回滚、多人协作或站点多主题切换。该边界避免引入低代码运行时、拖拽库或不受控代码执行，保证性能和可维护性。

现有技术教程、媒体库、网页实验室、登录、备份和网站设置保持原有职责。普通页面不替代教程文章；教程继续使用当前教程编辑与发布流程。

## 信息架构

后台新增“外观与页面”入口，提供三个独立区域：

| 区域 | 管理员可完成的工作 |
| --- | --- |
| 全局外观 | 选择受限的视觉令牌，查看即时缩略预览，保存或恢复默认外观。 |
| 首页编辑 | 开关首页模块、使用上下按钮改变模块顺序、编辑模块文案与文章显示数量。 |
| 页面管理 | 新建、编辑、预览、发布、下线和删除普通页面；决定是否加入顶部导航。 |

建议路由为：

- `/admin/appearance`：外观与页面总览，包含三个区域入口。
- `/admin/appearance/global`：全局外观。
- `/admin/appearance/home`：首页编辑。
- `/admin/pages`：页面列表与创建入口。
- `/admin/pages/[id]`：页面编辑。
- `/admin/pages/[id]/preview`：仅管理员可访问的草稿预览。
- `/pages/[slug]`：访客访问已发布的普通页面。

普通页面固定使用与教程阅读页一致的阅读模板。新页面若勾选“显示在顶部导航”，发布后自动出现在公开顶部栏；草稿和已下线页面绝不显示在导航、搜索或公开路由中。

## 模块架构

新增独立的 `site-designer`（或语义等价名称）模块，遵循项目既有分层方式：

- **领域层：** 定义外观令牌、首页模块配置、页面状态、保留地址和验证规则。
- **应用层：** 负责保存配置、恢复默认值、页面发布/下线、排序和导航筛选。
- **端口层：** 定义配置与页面仓储接口，避免页面组件直接依赖 Prisma。
- **适配层：** Prisma 仓储、Next.js 服务端操作和公开页面数据读取。
- **界面层：** 管理后台表单、实时预览、首页模块控制器和普通页面编辑界面。

`site-settings` 继续只持有博客名称、作者信息、基础导航和 SEO 等既有站点信息。`site-designer` 通过明确的公开读取接口向首页、顶部栏和普通页面提供已发布数据，避免让公开组件读取后台表单逻辑。

## 数据设计

新增三个彼此职责单一的持久化对象：

1. **AppearanceSettings（单例）**
   - `themePreset`、`accentPreset`、`backgroundTone`、`glassIntensity`、`radiusScale`、`buttonStyle`、`headerDensity`、`footerVisible`。
   - 字段均使用允许值枚举或受限数值，不存储原始 CSS、HTML 或 JavaScript。

2. **HomeLayoutSettings（单例）**
   - 每个首页模块的标识、启用状态、顺序和该模块允许的配置。
   - 初始模块为：首屏介绍、精选教程、最近文章、网页实验室、教程归档。
   - 文章数量仅允许安全范围内的整数；模块顺序不允许重复或未知标识。

3. **Page（普通页面）**
   - `id`、`title`、`slug`、`summary`、富文本 `content`、`coverMediaId`、`status`、`showInNavigation`、`seoTitle`、`seoDescription`、创建/更新时间、发布时间。
   - 状态只有 `DRAFT` 与 `PUBLISHED`；未发布页面不能公开访问。

普通页面封面仅引用现有媒体库资源。删除页面不会删除媒体库原文件。富文本复用现有文章编辑器的数据格式与能力，不增加第二套编辑器依赖。

## 管理后台体验

全局外观使用选择器、色板、分段控件和范围滑块，而不是代码输入框。右侧实时预览反映当前尚未保存的表单状态；保存后更新全站已发布外观。恢复默认操作必须二次确认。

首页编辑将模块排列为稳定列表。管理员通过可见性开关和上移/下移图标调整，而非引入拖拽依赖；模块展开后只显示其相关字段。保存时一次性校验整份首页配置，避免出现部分顺序已保存、部分文案未保存的状态。

页面管理列表显示标题、状态、更新时间和导航展示状态。编辑时自动从标题生成可修改的 `slug`；保存草稿、预览和发布是清晰分开的操作。离开含未保存修改的页面时显示离开确认提示。

所有新增界面使用中文 UTF-8 文案，并沿用现有公开网站的通透玻璃、浅色按钮和悬浮反馈。移动端控件应保持可点击且不发生文字溢出。

## 公开渲染与数据流

```text
管理员表单
  -> 应用层校验与保存
  -> AppearanceSettings / HomeLayoutSettings / Page
  -> 公开读取接口
  -> 首页、顶部栏、普通页面服务端渲染
```

首页读取当前外观与首页配置后，根据已启用模块和排序输出页面。全局视觉配置被转换为受控 CSS 变量或预定义类名，供公开网站和后台共用设计令牌；页面内容不可以覆盖这些全局样式。

顶部栏在读取原有导航配置后，追加已发布且标记为导航展示的普通页面。普通页面采用 `/pages/[slug]` 路径，避免与 `/tutorials`、`/labs`、`/archive`、`/about`、`/admin` 等现有路径冲突。

保存成功后，公开路由的缓存需要失效或重新验证，确保访客尽快看到更新后的首页、导航和页面内容。公开读取只返回已发布数据，不能通过猜测 ID 或 `slug` 取得草稿。

## 验证与错误处理

- 所有写入操作要求现有管理员会话；预览路由同样要求管理员会话。
- `slug` 必须唯一，使用安全字符，禁止空值，并拒绝已保留的公开或后台路径。
- 标题、摘要、SEO 字段和首页文案有明确长度上限；富文本由既有编辑器的受控输出处理。
- 外观字段、模块标识、排序和数量均在服务端验证；非法输入返回字段级中文错误，不写入数据库。
- 删除和恢复默认均需要二次确认；媒体引用不进行隐式删除。
- 数据迁移会生成与现有网站接近的默认外观和首页配置，升级后不会出现空白首页。

## 性能与部署

该功能只增加少量服务端代码、管理后台按需加载的界面代码和很小的 PostgreSQL 配置数据。公开页面继续服务端渲染，不新增拖拽框架、低代码引擎、画布、图表库或运行时执行器。现有 Docker Compose 部署流程不变，仅需包含数据库迁移。

预期 Docker 镜像大小仅随业务代码小幅变化，运行内存不会因页面搭建器或长期客户端状态而显著增加，适合 1C2G 服务器。

- `check:utf8`、模块边界检查、代码规范、类型检查和既有测试全部通过。
- 对外观配置允许值、首页模块排序、数量范围、`slug` 唯一性和保留路径编写单元测试。
- 对普通页面草稿不可公开访问、已发布页面可访问、导航展示筛选和首页模块渲染编写集成测试。
- 手动验收全局外观即时预览、保存后的公开首页更新、恢复默认、首页模块上下移动、普通页面草稿预览/发布/下线、中文界面和移动端文字不溢出。
- 构建 Docker 镜像并在 Docker Compose 环境中验证迁移、管理员登录、公开阅读和资源占用没有明显回归。

## 成功标准

管理员无需编写代码，即可在后台安全地统一调整网站视觉、编辑首页内容和模块顺序、创建及发布普通页面。公开页面与后台保持一致的视觉语言，现有教程和实验室功能不回归，部署成本仍适合轻量服务器。
